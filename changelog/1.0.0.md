# 1.0.0 (2026-06-23)

Changes since 0.4.0.

### Added
- Battery Percentage now uses the device-reported `percent_charged` when the API provides it, which is far more accurate than estimating from voltage (#11).
- New **WiFi Signal** sensor: WiFi signal quality in percent (`wifi_strength`).
- New **Last Seen** sensor: when the device last contacted the TRMNL server (`last_ping_at`). It needs no device-specific token.
- The voltage based battery fallback now uses a non-linear LiPo discharge curve instead of a linear map.
- Setup logs a warning if the entered API key does not start with `user_` (likely the wrong key).

### Changed
- Reworked README: prerequisites (Developer Edition add-on and the account API Key), installation, and configuration.

### Fixed
- Adding an already configured account now shows "already configured" instead of "unknown error" (#10).
- Opening the integration's Configure/Options no longer throws a 500 error (`config_entry` is read-only in current Home Assistant versions).
- Battery voltage is always parsed as a float before calculations.

### Removed
- **Breaking:** Removed the Device Access Token option and the Last Render sensor. The TRMNL API no longer returns the `rendered_at` field that sensor relied on. Use the Last Seen sensor (`last_ping_at`) for device liveness instead.
